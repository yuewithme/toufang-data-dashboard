# 投放数据看板

杭州沃虎科技有限公司投放数据看板，用于查看不同平台在选定日期范围内的消耗、客户量、环比变化和客户明细排行。

线上预览：[https://yuewithme.github.io/toufang-data-dashboard/](https://yuewithme.github.io/toufang-data-dashboard/)

## 功能概览

- 支持按消耗日期、排行数量、品牌名称、平台选择筛选数据。
- 日期选择支持当天、近七天、本月快捷范围。
- 左侧展示消耗数据统计和客户数据统计，包括本期、上期和环比变化。
- 中间展示近七天消耗趋势，按当前结束日期向前 6 天聚合源表消耗。
- 右侧展示小红书、视频号、支付宝三个平台的本期消耗、客户总量和客户排行。
- 原始数据页支持查看完整字段，并预留新增记录、修改记录、删除记录的后端操作逻辑。
- 当前版本使用 mock 源数据计算，页面所有汇总、趋势和榜单均由源数据聚合得出。

## 数据口径

后端接入前，页面使用 `src/lib/mock-data.ts` 中的 mock 数据作为源数据，并在前端完成筛选和聚合。mock 源表覆盖 2026-06-03 至 2026-06-16，且同一客户同一天可能有多条投放明细，便于模拟真实数据库按日汇总。

核心口径：

- 只统计 `客户群` 字段包含 `代投` 的记录。
- 消耗数据取源表字段 `非赠款消耗`。
- 近七天趋势以当前 `消耗日期` 结束日为最后一天，向前 6 天按 `消耗日期` 分组汇总 `非赠款消耗`。
- 平台字段由 `代理商账户名` 映射得到：
  - `支付宝-沃虎` -> `支付宝`
  - `小红书聚光乘风--沃虎` -> `小红书`
  - `小红书品专--沃虎` -> `小红书`
  - `腾讯视频号--沃虎` -> `视频号`

完整接口设计、字段映射和后端聚合规则见 [docs/api-contract.md](docs/api-contract.md)。

## 原始数据操作

原始数据页包含完整字段展示和记录选择能力。后端接入时需要支持：

- 新增记录：填写 `品牌名称`、`客户群`、`非赠款消耗`、`代理商账户名`。
- 修改记录：选中单条记录后，只允许修改 `客户群`、`非赠款消耗`、`代理商账户名`。
- 删除记录：不物理删除源数据，建议通过标记该记录不通过筛选，使其不再进入看板统计。
